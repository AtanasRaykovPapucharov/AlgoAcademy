# Naive search
* Test for match at every possible position
	* Simple but inefficient
	* Quadratic in time

<!-- section start -->

<!-- attr: {} -->
# Rabin-Karp
* Uses rolling hash
	* Linear in time
		* Quadratic if you check for collisions
* Capable of multiple pattern search
